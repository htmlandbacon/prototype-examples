ENV VARS

You can set varibles when your application is running, this can be very useful in a user research session, as you could adapt the data to the session.

if you look at /api/login you will see some fake details I have populated.

API_INVITE_KEY='TEST' gulp

You can also update API_NAME and API_DOB, if this was Heroku, you could update these the same way you add a username and password.

This can be found under the API routes folder.
